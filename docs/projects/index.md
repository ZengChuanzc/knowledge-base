# 🏗️ 企业级项目实践

> 从业务痛点出发，以技术方案落地。以下三个项目覆盖高并发性能优化、AI 工程化落地、大规模微服务架构设计，累计服务数十万用户，核心接口性能提升 80%~90%。

> 本文为真实企业项目实践，核心代码不涉及到项目源代码，仅提供思路方案。

## 项目一：赫兹运力平台系统

2026.04 - 2026.06 `Spring Boot` `Spring Cloud` `MySQL` `Redis` `RocketMQ` `Nacos` `Docker`

面向电力物资运输场景的企业级运力管理平台，覆盖 App、PC、小程序三端，支撑运营、司机、承运商多角色协同。系统累计处理 **400 万+ 运单**，是能源物流运输数字化的核心平台之一。

参与系统重构及新旧系统切换，主导报表中心异步化改造。针对高频报表查询接口，从 SQL 层到架构层逐层优化，将查询响应从 **1.2s 优化至 200ms 以内**；采用 RocketMQ 异步化 + 策略模式重构报表生成流程，实现业务解耦，解决接口长时间阻塞问题。

> **关键成果**：高频查询 1.2s → 200ms｜报表生成异步化｜系统平滑迁移

## 项目二：数字化移交通道管理平台

2025.07 - 2025.11 `Spring Boot` `Spring Cloud Alibaba` `MySQL` `Redis` `Qwen` `LangChain4j`

面向大型集团工程全生命周期移交场景，搭建外网业务通道 + 内网生产环境双模块分布式管理平台，解决建设、施工、监理、管理等多单位协同效率低、数据不规范、内外网数据隔离等行业痛点。

主导项目整体架构搭建，制定 Java 后端开发规范，统筹团队完成 **60% 以上核心功能开发与交付**。独立开发 AI Code Review 自动审查组件，基于 Git Webhook + LangChain4j + 私有化 Qwen 模型，结合 Embedding 检索与 Few-shot、否定指令等 Prompt 工程手段，将误报率从 **22% 降低至 9%**。基于 BPMS 开发多节点审核流程，重构跨单位审核业务逻辑，审核效率提升 **40%**。

> **关键成果**：AI 审查误报率 22% → 9%｜审核效率↑40%｜统筹 60%+ 核心功能

## 项目三：基建智慧工程管控项目

2022.10 - 2025.10 `Spring Boot` `Spring Cloud` `MySQL` `Redis` `Kafka` `RocketMQ` `Docker` `Jenkins`

服务 **30-40 万用户**的企业级基建工程管控平台，涵盖工程中心、进度中心、安全中心三大模块，支撑工程建设全流程管理、实时监控与海量数据采集。项目周期三年，是三个项目中规模最大、周期最长的。

缓存方面，针对 5 个高频接口（日均调用 120 万次），构建 **Redis + Caffeine 多级缓存**，平均响应从 **700ms 降至 100ms**，P99 从 **2.1s 降至 210ms**。数据处理方面，基于 Kafka 接收多省份设备监控数据，通过 **MyBatis 插件 + AOP 动态分表路由**解决单表数据膨胀瓶颈。架构方面，采用 **RocketMQ 异步消息驱动 + 补偿任务**替代同步调用，进度编制接口从 **1.5s 降至 80ms**。年度统计方面，通过 SQL 重写、索引优化及多线程并行计算，千万级数据量统计性能提升 **70%**。

> **关键成果**：700ms → 100ms｜1.5s → 80ms｜统计性能↑70%｜Kafka 动态分表

## 技术能力总结

| 能力维度 | 具体实践 |
|---------|---------|
| **性能优化** | 多级缓存（Redis + Caffeine）、SQL 重写与索引优化、多线程并行计算、Kafka + MyBatis 动态分表 |
| **架构设计** | 30-40 微服务治理、内外网双模块隔离、RocketMQ 异步消息驱动、补偿事务保障最终一致性 |
| **AI 工程化** | 大模型私有化部署（Qwen）、LangChain4j 集成、RAG + Embedding 检索、Few-shot Prompt 优化 |
| **工程规范** | CI/CD 自动化流水线（GitLab + Jenkins）、代码审查机制、架构规范制定、团队任务统筹 |
